# 架构决策 Playbook（项目落地层）

> 本文件包含**项目特定**的落地规则，随项目演进更新。
> 稳定原则 → 见 `architecture-meta-model.md`
> PR 决策模板 → 见 `architecture-checklist.md`

## 1. 本项目相态判断（上次 Review：2026-Q1）

整体处于**成长态**，按模块细分：

| 模块              | 相态   | 判断依据                           | 下次 Review |
| ----------------- | ------ | ---------------------------------- | ----------- |
| 用户/权限/租户    | 运营态 | 模型稳定 > 6 月，变更 < 2 次/月    | 2026-Q2     |
| 营销/金融/门店    | 成长态 | 需求确定性 ~60%，月变更 5-8 次     | 2026-Q2     |
| AI 相关功能       | 探索态 | 需求不确定，技术方案随模型能力变化 | 每月        |
| 基础设施（CI/CD） | 运营态 | 流程稳定，偶尔优化                 | 2026-Q2     |

**标注约定**：探索态代码必须标注 `// EXPLORATION: 此实现为快速验证，PMF 确认后需重构`

## 2. 风险信号监控

### 2.1 复杂性上升信号

| 信号                | 阈值               | Owner      |
| ------------------- | ------------------ | ---------- |
| 模块间依赖数增长    | 季度环比 > 20%     | 技术负责人 |
| 单个 Service 行数   | > 300 行           | 模块 Owner |
| 构造函数注入数      | > 6 个             | 模块 Owner |
| 同一 bug 跨模块重复 | ≥ 3 次             | QA         |
| 新人上手时间增长    | 比上一位新人 > 50% | TL         |

### 2.2 信号 → 行动 SOP

| 信号触发后                      | 响应时限             | 动作                 |
| ------------------------------- | -------------------- | -------------------- |
| 单个信号首次触发                | 当前 Sprint          | Sprint Review 中讨论 |
| 同一信号连续 2 个 Sprint        | 下一 Sprint 必须排期 | 创建修复任务         |
| 同一信号连续 3 个 Sprint 未解决 | 立即                 | 升级为 L3 决策流程   |

## 3. 渐进式验证机制

| 阶段 | 目标           | 范围           | 通过标准                          |
| ---- | -------------- | -------------- | --------------------------------- |
| PoC  | 验证技术可行性 | 独立分支       | 核心场景跑通 + 性能粗测           |
| 试点 | 验证业务适配性 | 1 个非核心模块 | 稳定运行 2 周 + 无 P0/P1 bug      |
| 全量 | 全面推广       | 所有目标模块   | 监控指标正常 + 团队反馈无重大问题 |

L1 不需要渐进验证；L2 视情况；L3/L4 必须。

## 4. 决策复盘机制

- 决策执行完成后 1 个月复盘实际效果 vs 预期效果
- 相态季度 Review 时回顾本季度所有 L3+ 决策
- 出现重大事故时分析是否与某个架构决策相关

案例库位置：`docs/design/decision-cases.md`。

## 5. AI 辅助角色定义

### AI 可以做

触发判断、约束发现、棱镜评估、方案对比、模板填写

### AI 不应做

最终拍板、相态判断、组织协作判断

## 6. 最小可行流程（团队 ≤ 3 人时）

| 步骤     | 简化做法                          |
| -------- | --------------------------------- |
| 触发判断 | 心里过一遍触发条件清单            |
| 相态判断 | 口头确认                          |
| 四棱镜   | 只回答轻量版 5 个问题             |
| 输出     | PR 描述或 commit message 中写结论 |
| 复盘     | 月度 1:1 时口头回顾               |

**底线**：L3+ 决策的结论必须有文字记录。
