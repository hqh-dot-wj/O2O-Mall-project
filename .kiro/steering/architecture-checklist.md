# 架构决策 Checklist（执行检查层）

> PR 提交、Code Review、上线前检查时使用。直接复制模板填写。
> 原则依据 → 见 `architecture-meta-model.md`
> 项目相态与信号 → 见 `architecture-playbook.md`

## 0. 快速入口：30 秒判断要不要走决策流程

回答以下问题，任一为"是"则需要走决策流程：

- [ ] 是否新增或替换了外部依赖？
- [ ] 是否新增了跨模块依赖？
- [ ] 是否变更了数据模型？
- [ ] 是否新增了应用或共享库？
- [ ] 是否涉及技术选型？
- [ ] 是否做了架构重构？
- [ ] 是否影响了多个应用？

**全部为"否"**：正常开发流程。**任一为"是"**：根据下方分级选择模板。

## 1. 决策分级

| 级别 | 特征                             | 使用模板          | 耗时预估   |
| ---- | -------------------------------- | ----------------- | ---------- |
| L1   | 单模块内部，不新增外部依赖       | 轻量版            | 5 分钟     |
| L2   | 新增跨模块依赖，或引入新工具库   | 轻量版            | 15 分钟    |
| L3   | 技术选型、架构重构、新增外部服务 | 完整版            | 30-60 分钟 |
| L4   | 影响多个应用                     | 完整版 + 方案对比 | 1-2 小时   |

## 2. 轻量版模板（L1 / L2）

```markdown
### 架构决策：[一句话简述]

1. **相态**：[模块名] 处于 [探索态/成长态/运营态]
2. **业务价值**：[这个改动对用户/业务的直接影响]
3. **最大风险**：[最可能出错的地方]
4. **退出成本**：[如果决策错误，回退需要多少成本]
5. **连接影响**：[是否新增跨模块依赖？方向是否符合架构图？]
```

## 3. 完整版模板（L3 / L4）

在 `docs/design/` 或 `apps/{app}/docs/design/` 创建独立文档：

```markdown
### 决策：[简述]

**相态判断**：涉及模块所处相态 + 四棱镜权重调整说明
**约束清单**：时间/人力/技术/兼容/合规/依赖约束
**业务价值棱镜**：价值链位置、技术 ROI、用户感知、机会成本
**时间棱镜**：短期收益、中期成本、长期锁定、半衰期评估
**能力棱镜**：团队掌握周期、单点风险、降级/退出方案
**复杂性棱镜**：显性成本、隐性成本、传染范围、退出成本
**结构影响**：新增/变更的模块边界和连接
**风险信号监控**：上线后关注的信号和触发回滚的阈值
**可逆性判断**：单向门/双向门、回退成本
**验证计划**（L3+ 必填）：PoC 范围、试点模块、全量条件
```

## 5. PR 自检清单

### 5.1 所有 PR 必检

- [ ] 代码符合项目规范（lint + typecheck 通过）
- [ ] 有对应的测试覆盖
- [ ] 内部包引用使用 `workspace:*`

### 5.2 涉及架构决策的 PR 额外检查

- [ ] 已判断决策级别（L1-L4）
- [ ] 已填写对应模板
- [ ] 跨模块依赖方向符合架构图
- [ ] 新增外部依赖已评估退出成本
- [ ] L3+ 决策有验证计划

### 5.3 数据模型变更的 PR 额外检查

- [ ] 已确认字段类型、默认值、索引
- [ ] 已评估对现有数据的迁移方案
- [ ] 已确认级联影响
- [ ] 已确认对前端类型的影响（需要重新生成 `@libs/common-types`）
