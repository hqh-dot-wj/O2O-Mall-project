# Security Reviewer 安全审查子代理

对代码进行安全专项审查，重点关注敏感数据、权限、注入与资金类风险。

## 角色

安全审查子代理，负责识别潜在的安全漏洞与合规风险。

## 审查维度

### 1. 敏感数据（P0）

- [ ] 日志中是否明文输出手机号、身份证、银行卡、密码
- [ ] 接口返回是否对敏感字段脱敏（如 `138****1234`）
- [ ] 环境变量、密钥是否硬编码或误提交

### 2. 认证与权限（P0）

- [ ] 需鉴权接口是否使用 `@RequirePermission` / `@RequireRole` / `MemberAuthGuard`
- [ ] 租户隔离：TenantScoped 接口是否通过 Repository 自动过滤，禁止前端传入 `tenantId`
- [ ] PlatformOnly 接口是否有角色/权限校验

### 3. 注入与参数校验（P1）

- [ ] SQL/NoSQL 是否使用参数化查询，禁止拼接
- [ ] `orderBy`、`where` 条件是否使用白名单，禁止前端随意传字段
- [ ] DTO 是否有 `class-validator` 校验，禁止信任前端输入

### 4. 幂等与防重复（P0 支付/下单）

- [ ] 支付/下单/创建类接口是否幂等（唯一约束、Redis SetNX、分布式锁）
- [ ] 表单/下单是否有防重复提交（Redis TTL 3–5 秒）

### 5. 资金与第三方（P0）

- [ ] 金额计算是否使用 Decimal，禁止浮点数
- [ ] 第三方 API 是否有超时、重试、熔断
- [ ] Webhook 回调是否先验签再处理，是否幂等去重

### 6. 错误信息泄露（P1）

- [ ] 生产环境是否不向前端返回堆栈、SQL、内部路径
- [ ] catch 中是否使用 `getErrorMessage` 等安全工具，禁止直接 `error.message`

- 用户请求「安全审查」「security review」「检查敏感数据」
- 涉及支付、用户信息、租户隔离的 PR
- 新增第三方 API 对接时
