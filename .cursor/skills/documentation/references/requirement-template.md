# 需求文档标准结构（11 章节）

```
1. 概述（1.1 背景、1.2 目标、1.3 范围）
2. 角色与用例（用例图、角色说明表）
3. 业务流程（3.1-3.3 核心流程，至少 3 个活动图）
4. 状态说明（4.1 状态机图、4.2 状态说明表）
5. 现有功能详述（5.1 接口清单、5.2 核心方法清单、5.3 业务规则详述）
6. 现有逻辑不足分析（6.1 P0、6.2 P1、6.3 P2、6.4 P3 级缺陷）
7. 市面主流系统对标（7.1 功能对比矩阵、7.2 差距总结，对标 3-4 个竞品）
8. 跨模块缺陷（X-1、X-2...）
9. 验收标准（9.1 现有功能验收、9.2 待修复验收）
10. 演进建议与待办（10.1-10.3 分阶段）
11. 非功能需求（11.1 性能、11.2 可靠性、11.3 安全性、11.4 可维护性）
```
