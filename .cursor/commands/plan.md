# 任务规划

模糊需求时拆解任务，或结构化需求时生成任务列表。

## 输入

- 模糊需求描述，或 @文档（PRD、Notion 等）

## 执行步骤

1. **若需求模糊**：输出澄清清单（目标、范围、现有参考、边界条件）
2. **若需求已结构化**：按 task-execution 拆解为可执行任务列表
3. **实现前含方案探索**：涉及数据量、缓存、并发、一致性时，先列出 2–3 种方案及优劣，再选型

## 输出格式

### 澄清模式

- 待澄清问题 1、2、3…
- 建议用户补充的信息

### 任务模式

- [ ] 任务 1（依赖：无）
- [ ] 任务 2（依赖：任务 1）
- …

## 与 task-execution 的关系

- 本命令产出任务列表后，可交由 task-execution Skill 执行
- 增量模式：若部分任务已打勾，只执行未完成项
