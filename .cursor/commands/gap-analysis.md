# Gap 分析

需求文档 + 部分实现时，分析已完成项、待完成项、冲突项。

## 输入

- @需求文档（PRD、Notion 页、或用户描述）
- 可选：限定路径（如 `apps/backend/src/module/xxx`）

## 执行步骤

1. **读需求**：提取需求中的功能点、接口、页面、流程等清单
2. **语义搜索**：在指定路径（或全项目）内搜索相关代码
3. **逐项对比**：将需求清单与代码实现逐项对照
4. **输出表格**：按「已完成」「待完成」「冲突」分类

## 输出格式

| 需求项 | 状态   | 说明               |
| ------ | ------ | ------------------ |
| xxx    | 已完成 | 对应文件/接口      |
| xxx    | 待完成 | 缺失内容           |
| xxx    | 冲突   | 需求与实现不一致处 |

## 依赖顺序（跨模块时）

- 后端 → `pnpm build` → `pnpm generate-types` → 前端
- 只补缺口，不重写已完成部分；若发现 bug/安全/逻辑错误应修复，并同步更新前端及测试
